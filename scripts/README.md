# Scripts

Repository automation and local developer helpers.

- `admin/`: auth and admin maintenance helpers
- `agents/`: agent orchestration helpers
- `build/`: packaging and release helpers
- `dev/`: local development utilities and proxies
- `evals/`: dataset preparation and benchmark runners
- `i18n/`: translation validation and repair helpers
- `research/`: experiments and one-off investigation scripts
- `windows/`: Windows-specific maintenance helpers

Only leave files at the root of `scripts/` when they are genuinely shared entry
points that do not fit one category.
