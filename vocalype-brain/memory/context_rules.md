# Context Rules

- The model's general knowledge is allowed only as reasoning support.
- Facts about Vocalype must come from retrieved memory, repo inspection, metrics, or user-provided data.
- If the model has not retrieved evidence, it must say confidence is low.
- Do not invent Vocalype architecture details.
- Do not recommend unrelated projects.
- Always optimize for Vocalype product, distribution, conversion, retention, or revenue.
- Every action must include metric, validation test, risk, and next step.
- Context window is limited, so retrieve only relevant chunks.
- Prefer concise context over huge prompts.
- The founder remains final authority.
- Autonomous thinking, controlled execution.
