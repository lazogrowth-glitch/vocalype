# internal/brain/patches/

This directory contains Safe Patch files generated by Vocalype Brain V3.

---

## What Is a Safe Patch?

A safe patch is a **Markdown proposal file** â€” not an executable diff.

It describes a change that Brain recommends but does **not** apply automatically.

Every patch file must be reviewed and approved by the founder before any code is touched.

---

## Safety Classes

| Class | Meaning | Auto-apply allowed? |
|---|---|---|
| `brain_safe` | Only `internal/brain/` files are targeted | No â€” always review first |
| `docs_safe` | Only README/documentation/markdown files | No â€” always review first |
| `product_proposal_only` | Product code is involved | **No â€” never auto-apply** |
| `unsafe` | Forbidden scope detected | No patch generated |

---

## File Naming

```
patch_YYYYMMDD_HHMMSS_<slug>.md
```

Example:
```
patch_20260424_091500_measure_activation_failure.md
```

---

## Patch File Sections

Each patch file includes:

1. Title, safety class, task type, manual approval flag
2. Target files
3. Reason
4. Proposed changes (or "PROPOSAL ONLY" notice)
5. Exact implementation prompt
6. Validation plan
7. Risks
8. Rollback plan
9. Safety rules

---

## Rules

- **Do not apply `product_proposal_only` patches automatically.**
- **Do not apply any patch without reading it first.**
- Always run `npm run lint` and manual test scenarios after a product patch.
- Revert with `git checkout -- <file>` if validation fails.
- Never bypass commit hooks (`--no-verify`).
- Never apply to: `backend/`, `src-tauri/`, `src/lib/auth/client.ts`, `src/lib/license/client.ts`, payment, billing, security, or translation files.
